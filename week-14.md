# 第14-16周：团队综合项目

---

## 📋 本周基本信息
- **学时**：8学时（第14-16周）
- **实验类型**：研究创新性实验
- **前置知识**：前面所有实验内容

---

## 🎯 本周学习目标

通过团队综合实验，使学生：

### 知识目标
- 掌握团队协作开发的基本流程和方法
- 掌握 GitHub 团队协作功能（Pull Request、Issue）
- 掌握项目文档编写和展示的方法

### 能力目标
- 能够以小组为单位开发一个完整的跨平台 APP 和 Web 个人网站
- 能够进行简单的产品设计和原型设计
- 能够绘制架构图和流程图
- 能够使用 GitHub 进行团队协作开发
- 能够进行项目展示和答辩

### 素质目标
- 培养团队协作和沟通能力
- 培养项目管理和时间管理能力
- 培养产品思维和用户体验意识
- 培养科技创新和知识产权意识
- 培养职业道德和社会责任

## 🛠️ 实战任务

### 任务一：项目规划
- 组队（3-4人一组）
- 选题与需求分析
- 产品设计与原型制作
- 项目计划与任务分配

### 任务二：团队协作开发
- GitHub 仓库创建与权限管理
- 分支策略与代码规范
- Pull Request 代码审查
- Issue 跟踪与讨论
- 里程碑管理

### 任务三：项目开发
- Flutter APP 开发
- Web 网站开发
- 功能测试与 Bug 修复
- 持续集成与部署

### 任务四：项目展示
- 项目演示准备
- 文档撰写（README、设计文档、用户手册）
- 项目展示与答辩
- 互评与成绩评定

## ⚠️ 常见问题
- 团队协作不顺畅？明确角色分工，多沟通，善用 GitHub Issues
- 项目进度滞后？及早发现问题，调整计划，寻求老师帮助
- 代码冲突频繁？合理划分模块，频繁提交，及时合并
